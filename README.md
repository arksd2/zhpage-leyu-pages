# zhpage-leyu-pages

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面以静态网页形式存放，可通过 GitHub Pages 等服务直接访问。项目不针对特定域名或网站，而是作为一个通用页面集合存在。

## 目录说明

```
zhpage-leyu-pages/
├── index.html          # 主入口页面（可选）
├── pages/              # 页面归档目录
│   ├── example1.html   # 示例页面1
│   ├── example2.html   # 示例页面2
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

`pages/` 目录用于存放独立的 HTML 页面，每个页面可包含其所需的样式与脚本。`assets/` 目录存放共用的静态资源文件，供页面引用。

## 页面归档说明

归档的页面均为独立、自包含的 HTML 文件，可直接在浏览器中打开运行。部分页面可能依赖 `assets/` 目录下的资源文件，建议保持目录结构完整以确保正常显示。

页面内容可能涉及多种用途，例如信息展示、工具页面、存档页面等。每个页面均以清晰的文件名命名，便于识别和管理。

## 维护说明

- 添加新页面时，请将 HTML 文件放入 `pages/` 目录，并确保引用资源路径正确。
- 若页面使用外部资源（如字体、库文件），建议下载至 `assets/` 目录，避免依赖外部链接。
- 更新现有页面时，请保留原始文件名，以便保持链接稳定性。
- 本仓库不主动推广或宣传任何页面内容，仅作为静态页面托管与归档使用。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在本地打开 `pages/` 目录下的 HTML 文件，或通过 HTTP 服务器访问。
3. 若启用 GitHub Pages，可将仓库根目录或 `docs/` 目录作为发布源。
